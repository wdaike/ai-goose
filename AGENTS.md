# AGENTS Instructions

goose is an Electron desktop frontend for OpenAI's Codex app-server. There is
no Rust code and no backend of our own.

## Architecture

The Electron main process spawns `codex app-server` (the official binary, from
`PATH` or `GOOSE_CODEX_BIN`) and bridges its stdio JSONL JSON-RPC to the
renderer over IPC (`ui/desktop/src/codex/bridge.ts`, exposed as
`window.codex`). The renderer speaks the app-server v2 protocol directly.

Codex owns everything backend-shaped: inference, tool execution, the MCP
client, prompt assembly (including `AGENTS.md` discovery), conversation
storage (`thread/*`), skills, config (`config/read`, `config/batchWrite`),
model catalog (`model/list`), and accounts. Don't reintroduce a parallel
implementation of anything Codex already provides — route to it instead.

Key renderer modules:

- `src/codex/client.ts` — typed request façade over `window.codex`
- `src/codex/protocol/` — TS types generated by `codex app-server generate-ts`
- `src/codex/engine/controller.ts` — chat engine: drives `thread/start`,
  `turn/start`, `turn/steer`, `turn/interrupt`, approvals; implements the
  `AcpChatSessionController` interface the UI consumes
- `src/codex/engine/mapItems.ts` — maps Codex thread items to the UI's
  `Message` model (agentMessage→text, reasoning→thinking,
  commandExecution/mcpToolCall/fileChange→toolRequest/toolResponse)
- `src/acp/` — legacy data-layer modules, now codex-backed or stubbed; the
  chat store (`chatSessionStore.ts`) and permission plumbing are still live

## Commands

```bash
cd ui/desktop && pnpm install
cd ui/desktop && pnpm run start-gui   # run the app (needs `codex` on PATH)
cd ui/desktop && pnpm run typecheck
cd ui/desktop && pnpm test
cd ui/desktop && npx eslint src --ext .ts,.tsx
```

To regenerate protocol types after upgrading codex:

```bash
cd ui/desktop && codex app-server generate-ts --out src/codex/protocol
```

## Structure

```
ui/desktop/               # Electron app (the whole product)
├── src/main.ts           # Electron main; spawns codex app-server
├── src/codex/            # codex bridge, protocol types, chat engine
├── src/acp/              # legacy seams, codex-backed or stubbed
└── src/components/       # UI (unchanged goose look & feel)
```

## Rules

- Codex: All backend behavior belongs to `codex app-server` — extend
  `src/codex/engine/` rather than reimplementing anything Codex provides
- MCP: servers are configured in codex `config.toml` (`mcp_servers.*`) via
  `config/batchWrite`; status/tools via `mcpServerStatus/list`
- UI Desktop: Use local `src/types/*` types. Do not import generated OpenAPI
  types/client code from `ui/desktop/src/api`
- UI prefs that have no codex home live in `localStorage` (`src/acp/config.ts`)

## Code Quality

- Comments: Write self-documenting code - prefer clear names over comments
- Comments: Never add comments that restate what code does
- Comments: Only comment for complex algorithms, non-obvious business logic, or "why" not "what"
- Simplicity: Don't make things optional that don't need to be
- Simplicity: Booleans should default to false, not be optional
- Simplicity: Avoid overly defensive code

## Never

- Never: Recreate `ui/desktop/src/api` or add `@hey-api/openapi-ts` to `ui/desktop`
- Never: Hand-edit files under `src/codex/protocol/` — they are generated
- Never: Reintroduce a goose-owned backend (ACP server, message store, MCP client)

## Entry Points

- UI: ui/desktop/src/main.ts
- Codex bridge: ui/desktop/src/codex/bridge.ts
- Chat engine: ui/desktop/src/codex/engine/controller.ts
